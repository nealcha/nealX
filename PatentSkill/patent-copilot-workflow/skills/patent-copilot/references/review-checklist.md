# Review Checklist

## Topic Fit and Scope

Check whether:

1. The selected topic is supported by the sample PDFs or research notes.
2. The scheme is centered on a structural solution suitable for a utility model.
3. The draft avoids drifting into a pure method or software-only description without structural carrier.
4. The scope is focused enough to form one coherent filing package.

## Disclosure Structure

Check whether the draft clearly covers:

1. Name
2. Technical field
3. Background or existing problems
4. Purpose
5. Main structural content
6. Beneficial effects
7. Figure descriptions
8. Specific implementation

## Figure and Numbering Consistency

Check whether:

1. Each key component has a stable identifier.
2. The same number does not point to different parts in different places.
3. Figure themes are distinct.
4. Figure descriptions match the declared purpose of each figure.
5. Component definitions in the body match what the figures appear to show.

## Patent Package Closure

Check whether:

1. The reviewed draft resolves the high-priority issues found in the review pass.
2. The review report is severity-ordered and points to concrete fixes.
3. The unresolved questions are explicit instead of being silently ignored.
4. The figure prompts still match the reviewed draft after revision.

## Domain Accelerator Quality

Check whether the accelerator page:

1. Explains the core structure in plain language.
2. Separates explanation from formal filing text.
3. Highlights easy-to-confuse parts.
4. Helps a non-specialist review the output.

## Human Confirmation List

Always call out items that require human confirmation:

1. Actual novelty or protectability
2. Legal risk
3. Final wording for formal filing
4. Whether the figures are good enough for submission
5. Whether measured or estimated performance statements are acceptable
